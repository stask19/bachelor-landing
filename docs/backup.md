# Стратегія резервного копіювання (Backup Strategy)

Для забезпечення надійності даних системи оренди автомобілів впроваджено наступну політику:

## 1. Резервне копіювання коду

- Вихідний код системи постійно версіонується у хмарному репозиторії GitHub.
- Усі зміни проходять через коміти з описом виконаної роботи.

## 2. Резервне копіювання бази даних

- **Частота:** На етапі MVP історія запитів не зберігається, але після підключення SQLite/PostgreSQL налаштовуються щоденні дампи бази даних о 03:00 ночі.
- **Зберігання:** Дампи архівуються (tar.gz) та відправляються на зовнішнє хмарне сховище.
- **Термін зберігання (Retention):** Щоденні копії зберігаються 7 днів, щотижневі 1 місяць.
